# 需求概要设计生成（基于 OpenSpec PDF 模板）

## 适用场景（触发条件）

- 需要把"需求/PRD/用户故事/口头描述"整理为**需求概要设计**。
- 输出必须**以 `assets/template.pdf` 为模板**，并以**简体中文**撰写。

## 强制约束

- 必须使用并遵循仓库内模板：`assets/template.pdf`。
- 所有内容用简体中文；术语保持一致，避免中英混用（专有名词除外）。
- 信息缺失时，先做**最小假设**并在文中显式标注"【假设】…"。同时输出"待确认问题清单"。
- 原始文档中的超链接必须保留。
- 原始文档中的图片、图表必须保留，图片统一放在 images 目录下，在最终输出的 md 文件中引用。

## 输入要求（尽量从仓库检索，不足再追问）

优先从当前变更目录、需求文档、接口文档、issue/任务描述中提取。

最小输入集合：
- 需求背景与目标（1-2 段）
- 目标用户/使用场景
- 核心流程（主路径）
- 功能清单（含优先级）
- 约束（端/版本/依赖/性能/安全/合规）

## 工作流（生成步骤）

1. 定位模板：读取并确认模板路径为 `assets/template.pdf`。
2. 提取模板结构：
   - 若无法直接解析 PDF 内容，则以"章节映射策略"填充：输出内容按模板的**章节顺序**组织；章节标题沿用模板标题；模板缺失标题则使用通用章节集（见下）。
3. 归纳需求：把输入信息整理为结构化要点（目标、范围、非目标、用户故事、验收口径）。
4. 形成概要设计：
   - 业务/交互流程（文字+Mermaid 流程图或 Mermaid 源代码）
   - 数据/对象模型（字段、约束、来源）
   - 接口与集成（入参/出参/错误码/幂等/重试）
   - 状态与异常（边界条件、降级、回滚）
   - 非功能（性能、可用性、安全、隐私、审计、可观测性）
5. 产出"待确认问题清单"与"风险清单"。
6. 最终输出：生成一份**可直接粘贴/导出到模板 PDF 的中文内容稿**，并同步保存到当前目录，文件名: req-outline-output.md，文件已存在则覆盖。

## 章节映射策略（当无法读取 PDF 时使用）

按下列顺序输出（并尽量与模板章节对齐）：
1. 概述（背景、目标、范围、非目标）
2. 术语与缩略语
3. 需求说明（用户画像/场景、用户故事、功能列表、优先级）
4. 总体方案（架构/模块拆分、关键设计决策）
5. 详细设计（流程、数据模型、接口、状态机、异常处理）
6. 非功能性需求（性能/容量、可靠性、兼容性、安全与隐私、可观测性）
7. 验收标准与测试建议
8. 风险与应对
9. 待确认问题

## 输出要求（必须包含）

- 标题、版本、作者、日期（若缺失则用【假设】占位）
- 功能范围与非目标
- 总体方案必须输出整体流程 mermaid 架构图或Mermaid 源代码
- 详细设计必须输出模块级别 mermaid 架构图或Mermaid 源代码
- 关键流程（主路径 + 至少 2 个异常/边界）
- 数据/接口关键字段与约束
- 风险与待确认问题
