## 1 引言
### 1.1 背景
AI字幕（Real-time Subtitle）面向通讯与平板产品线，通过端云协同的 ASR/MT 能力实现实时语音转写、翻译与字幕渲染，以满足全球用户在会议、影音、学习等场景下的跨语言沟通需求，目标上线时间为 2024 年 8 月。

### 1.2 术语与缩略语
| 术语 | 说明 |
| --- | --- |
| ASR | Automatic Speech Recognition，自动语音识别，将语音转换为文本 |
| TTS | Text-to-Speech，将文本转换为语音 |
| MT | Machine Translation，机器翻译，实现源文本到目标文本的转换 |
| SDK | Software Development Kit，软件开发工具包 |

### 1.3 参考资料
- Ergo：TCL AI_v8.2441.19_Ergo.pdf（Page 81-90、125-128）
- 微软 ASR+TTS API 参考仓库及文档

## 2 需求分析
### 2.1 需求概述
#### 2.1.1 需求背景
- 全球内容创作者与用户希望快速获取不同语言字幕；
- 手机/平板在会议、观影、网课等场景对实时字幕的诉求显著；
- 需支持在线/离线模式、端云切换，以及数据安全合规。

#### 2.1.2 需求列表（节选）
1. 侧边栏、Quick Settings、TCL AI 入口新增字幕按钮，并保持状态同步。
2. 字幕浮窗实时展示转写/翻译结果，提供历史滚动、发言断句、拖动与尺寸调整。
3. 支持本地/云端 ASR，自动切换至云端翻译；断网时提供本地降级提示。
4. 提供翻译语种设置、下载、搜索、双语显示、字号/透明度预览。
5. 提供字幕复制与 Notes 保存提醒（含保存/不保存/取消三选项）。

#### 2.1.3 需求范围与限制
- 平台：Android V 及以上版本，手机与平板；
- 市场：海外项目为主，依赖 Google 翻译 SDK；
- 硬件：需支持多芯片（MTK/QCOM/展锐），适配横竖屏与分屏；
- 权限：需登录 AI 账号及授权麦克风、悬浮窗、存储等权限。

#### 2.1.4 需求整体目标
1. 高质量翻译（准确、自然）；
2. 多语言覆盖并可持续扩展；
3. 实时/离线翻译能力；
4. 多场景应用（语音输入、媒体音频、会议、课堂等）；
5. 优秀用户体验（易用、可个性化、跨设备一致）；
6. 数据安全与隐私合规（GDPR 等）；
7. 可扩展的技术架构（便于功能与性能演进）。

#### 2.2.4 设计约束分析
- 常驻/动态内存增量需在评估范围内，必要时与性能 SE 评审；
- 软件平台：Android V+，支持 MTK/QCOM/展锐；
- 功耗：优化麦克风与 ASR 使用，后台任务受控；
- 安全合规：数据加密、权限最小化、GDPR；
- 三方依赖：Google 翻译 SDK、微软 ASR/TTS 模型需统一管理。

#### 2.2.5 依赖分析
- 外部：AI 能力服务（微软 ASR、Google MT）、TCL 云模型下载、BI 埋点服务；
- 内部：UI 模块、核心服务模块、数据存储模块、监控日志模块，互相依赖以完成字幕渲染、设置持久化、日志上报。

#### 2.2.6 影响分析
- 新增全局浮窗/分屏入口需核对全局弹框策略，避免与既有功能冲突；
- 需评估对系统性能、功耗、内存的影响，并配合产品/平台 SE；
- 关于 Notes 存储与复制功能，需确认用户数据合规与多端同步策略。

## 3 总体架构设计
### 3.1 零层设计
Subtitles 作为系统内置（不可卸载）应用，位于系统多入口生态中，需与控制中心、Sidebar、Settings、TCL AI 等模块协同，同时访问 TCL Cloud 模型与 Notes 应用。

### 3.2 一层设计
#### 3.2.1 系统总体结构（文字说明）
- **UI 层**：侧边栏入口、Quick Settings、字幕浮窗/分屏界面、设置面板、保存弹框；
- **网域层**：会话控制、状态机（开关、识别、翻译、异常）、权限与登录管理；
- **数据层**：ASR/MT 结果缓存、用户设置（DataStore）、Notes 存储、语言包下载状态；
- **外部服务层**：AI 能力服务（ASR/MT）、TCL 云语言包、BI 埋点。

#### 3.2.2 场景时序 / 关键流程
1. 入口点击 → 权限/登录校验 → 初始化 ASR/模型 → 打开字幕浮窗；
2. 音频采集（媒体/环境） → ASR Recognizing 流 → 字幕逐字渲染 → ASR Recognized 触发整句校对 → 可选 MT 翻译 → 双语显示；
3. 网络异常 → 提示并切换本地翻译 → 记录日志 → 恢复后回切云端；
4. 关闭字幕 → 判断是否有 Notes → 弹框引导保存/退出；
5. 分屏/拖动/尺寸调整 → UI 限位与位置记忆。

#### 3.2.3 模块分解矩阵（概略）
| 功能点 \ 模块 | UI 模块 | AI 能力模块 | 数据存储模块 | 监控日志模块 |
| --- | --- | --- | --- | --- |
| TCL AI 入口 | ✓ |  |  | ✓ |
| Quick Settings | ✓ |  |  | ✓ |
| Sidebar | ✓ |  |  | ✓ |
| 设置（语言/字号/透明度） | ✓ | ✓ | ✓ | ✓ |
| 麦克风/音源选择 | ✓ | ✓ |  | ✓ |
| 拖动条 / 分屏 / 浮窗 | ✓ |  | ✓ | ✓ |
| 复制/Notes 保存 | ✓ |  | ✓ | ✓ |

### 3.3 开发和运行环境
- 开发：Windows + Android Studio；
- 测试：多芯片真机、弱网/断网场景、横竖屏/分屏。

### 3.4 模块开发方式
- 新开发模块，采用模块化并行方式；可视场景拆分为 UI、服务、数据子模块。

### 3.5 技术方案风险点与应对
| 风险 | 详情 | 级别 | 应对 |
| --- | --- | --- | --- |
| AI 模型体积与版本 | 多语模型可能导致 APK 体积膨胀 | 中 | 统一接入 AI 能力服务模块，集中管理/下载 |
| 外部 SDK 升级 | ASR/MT SDK 接口变更 | 中 | 采用版本号管理，回归测试 |
| 悬浮窗/拖拽/分屏适配 | 不同分辨率/机型表现 | 低 | 前台服务 + 系统 API，阻尼与限位策略 |
| 整句校对实现 | 需要双通道输出 | 中 | 使用 Recognizing/Recognized 双接口并结合标点断句 |
| 入口状态同步 | 多入口开关一致性 | 低 | 系统值统一存储，入口监听 |

## 4 二层设计（节选）
### 4.1 语言管理模块
- **结构**：接口层（供 UI 调用）、语言列表管理、下载管理、设置持久化、云模型适配。
- **流程**：获取支持语言 → 显示下载状态 → 用户搜索/选择 → 未下载则触发下载 → 更新 DataStore → 回显 UI。
- **时序**：UI 请求语言列表 → 模块查询缓存/云 → 返回列表；用户切换语种 → 调用 AI 能力服务下载 → 回调状态 & 更新设置。
- **说明**：模块以服务形态存在，暴露获取/更新接口并写入 DataStore。

### 4.2 主页面模块
- **结构**：入口控制、浮窗渲染、识别控制、分屏管理、历史记录。
- **流程**：
  1. 开启流程：入口点击 → 校验权限 → 初始化服务 → 展示浮窗；
  2. 识别流程：音源采集 → ASR 流输出 → 渲染字幕 → 整句校对替换；
  3. 分屏/浮窗切换：按钮触发 → 调系统 API → 状态同步；
  4. 关闭流程：X 按钮 → 判断 Notes → 弹框引导。
- **时序**：入口 → Controller → AI 能力服务 → UI 渲染 → Notes/复制组件。

### 4.3 设置模块
- **结构**：效果预览、翻译配置、字号/透明度控件。
- **流程**：打开设置 → 读取当前设置 → 用户调整 → 实时预览 → 确认保存/取消还原。
- **说明**：作为 Compose/Fragment 组件存在，读写 DataStore。

### 4.4 语言设置模块
- **结构**：语言列表展示、搜索、下载进度显示；
- **流程**：点击语种 → 若未下载则下载 → 完成后自动选中；
- **说明**：与语言管理模块共享数据层，UI 层组件化实现。

## 5 接口设计
### 5.1 提供接口
- **内部接口**：
  - DataStore 读写接口：`readSetting(key)` / `writeSetting(key,value)`；
  - 设置状态接口：封装 DataStore，实现批量更新与监听；
  - 字幕内容导出接口：供复制/Notes 保存调用；
  - 语言管理接口：`getSupportedLanguages()`、`updateLanguageSetting(source,target)`、`downloadLanguage(langCode)`。
- `【假设】` 当前未对外开放 SDK，如未来沉淀 CBB，再补充外部接口章节。

### 5.2 依赖接口
| 接口 | 描述 | 入参 | 出参 |
| --- | --- | --- | --- |
| 模型初始化 | AI 能力服务加载模型 | 源语、目标语 | 初始化状态 |
| 识别翻译接口 | AI 能力服务识别音频并翻译 | 音频流、源语、目标语 | 实时/整句文本 |
| 模型信息接口 | 查询语言包下载状态 | 语言信息 | 下载/就绪状态 |
| 模型下载接口 | 下载语言/语音包 | 语言列表 | 下载进度/结果 |
| BI 埋点接口 | 输出埋点日志 | 埋点事件 | 上报结果 |
| 分屏接口 | 系统分屏调用 | 分屏参数 | 执行结果 |

## 6 非功能设计
### 6.1 性能
- 识别/翻译响应 1-2s；
- 控制新增进程/线程，必要时走评审；
- 监控字幕渲染、分屏切换、开启关闭的耗时。

### 6.2 功耗
- 环境麦克风默认关闭，仅在用户开启时采集；
- 云端请求失败及时降级；
- 后台任务（模型下载、Notes 保存）需节能策略。

### 6.3 内存
- 常驻/动态内存增量需量化并与性能 SE 确认；
- 通过懒加载模型、释放历史缓存控制占用；
- Notes 保存完成后及时清理内存内容。

### 6.4 可靠性（DFR）
- 登录异常：弹窗引导登录并关闭字幕；
- 网络异常：切换本地翻译并提示；
- 敏感词：Toast 高亮提示；
- 分屏不支持：Toast “当前应用不支持分屏”；
- Notes 缺失：关闭时提示是否直接退出。

### 6.5 安全性（DFSe）
- 语音/字幕数据传输加密；
- 语音采集与 Notes 保存前提示权限与隐私；
- 严格遵守 GDPR，记录隐私声明与场景；
- 控制访问权限，防止未授权模块访问字幕内容。

### 6.6 可维护性
- 代码模块化、接口化，遵循编码规范；
- 完整文档与注释；
- 引入自动化测试与持续集成，保障回归效率。

### 6.7 可测试性（DFT）
- 提供 ADB 命令/日志开关，输出识别/翻译状态；
- 支持测试 APK 或调试开关模拟异常；
- 埋点与日志支持定位问题。

### 6.8 可服务型性（DFS）
- 接入公司 BI 埋点，记录入口点击、识别状态、异常、Notes 保存等；
- 可通过 OTA / 应用更新推送修复。

### 6.9 可配置性（DFCu）
- 通过 Feature Flag/FCM 控制功能开关与子功能（如 Notes 保存、双语模式）；
- 针对不同区域/机型配置差异化（如语种列表、云端服务）。

### 6.10 兼容性（DFC）
- 支持 Android V+，兼容 MTK/QCOM/展锐；
- 横竖屏、分屏、悬浮窗、多窗口场景适配；
- 确保旧机型与新版本之间的行为一致或可配置。

### 6.11 解耦
- UI、AI 能力、数据存储、日志模块通过接口解耦；
- 后续可沉淀为 CBB 模块（如语言管理服务）。

### 6.12 全球化
- 支持主流语种（含多地区英文、法语、西语等）；
- 语言列表按系统语言排序，支持 A-Z 搜索；
- 考虑本地化翻译质量与语料持续扩展。

## 7 数据结构设计
- **用户设置 DataStore**：`sourceLang`、`targetLang`、`dualLineEnabled`、`fontLevel`、`opacityLevel`、`windowPosition`、`windowSize` 等；
- **语言包状态**：`langCode`、`langName`、`downloadStatus`、`progress`、`modelType`；
- **字幕内容缓存**：按时间戳/发言人存储，支持复制与 Notes 写入；
- **Notes 存储**：标题格式 `AI字幕 yyyy-MM-dd HH:mm`，内容包含完整字幕文本。

## 9 验收标准与测试建议
1. 功能：多入口开关、实时字幕与翻译准确性、历史滚动、发言断句、分屏/浮窗切换、复制与 Notes 保存流程、异常提示。

## 10 风险与应对
| 风险 | 说明 | 应对措施 |
| --- | --- | --- |
| AI 模型体积/版本管理 | 语言包多导致 APK 或缓存膨胀 | 统一 AI 能力服务模块管理下载/缓存，按需加载 |
| 外部 SDK 升级 | 接口兼容风险 | 采用版本号治理、回归测试、监控异常 |
| 悬浮窗/分屏适配 | 多分辨率/机型差异 | 前台服务 + 系统 API + 限位与阻尼策略 |
| 整句校对实现 | 依赖双输出接口 | 使用 Recognizing/Recognized 流 + 标点断句策略 |
| 入口状态同步 | 多入口状态不一致 | 统一状态中心（系统值/DataStore）+ 监听回调 |
| Notes 依赖 | Notes 未安装/权限不足 | 提供仅复制路径，弹框提示保存失败 |
